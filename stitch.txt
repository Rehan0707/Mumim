GOOGLE STITCH MASTER PROMPT — MUNIM.AI FRONTEND DESIGN

PROJECT OVERVIEW:
Munim.ai is a WhatsApp-first AI Business Operating System for local Indian shops, built for the Takeover'26 hackathon. It enables shop owners to sell products, track stock, and manage customers through WhatsApp — with zero app install required from customers.

The project is a monorepo with a React + TypeScript + Vite + Tailwind CSS frontend and a FastAPI Python backend. The frontend communicates with the backend via REST API endpoints and WebSocket for live updates.

DESIGN GOAL:
Create a clean, modern, professional UI that mirrors the WhatsApp color theme — white and light green (#128c7e teal as primary, #dcf8c6 light green as accent, #f0f2f5 light gray background). The design should feel calm, approachable, and trustworthy for non-technical shop owners in India.

NAVIGATION STYLE — CONTINUITY & COMPACTNESS:
Both landing page and dashboard use an identical pill-shaped (Dynamic Island style) floating navbar:
- Centered horizontally, floating at the top with subtle shadow
- White background with rounded-full shape
- Compact padding — only as wide as the content it holds
- Same visual style across all pages for continuity
- On mobile: hamburger menu with dropdown

LANDING PAGE SECTIONS:
1. Navbar: Floating pill with MunimAI brand name, Sign in button, Dashboard button
2. Hero: Headline, description text, CTA buttons, right side has dashboard preview card
3. Why section: 3 column cards explaining the value proposition
4. Flow section: 4 column step cards showing the message flow (01-04)
5. Views section: 2 column grid with feature panels
6. Preview section: KPI cards and order flow visualization
7. CTA section: Final call to action with design direction panel
8. WhatsApp preview within hero: Chat bubbles in real WhatsApp colors (bg #ece5dd, inbound white, outbound #dcf8c6)

DASHBOARD PAGES:
1. Home/Overview: Top row KPI stat cards (Revenue, Orders today, Pending, Customers), Recent orders list, Live feed activity panel, Action items, Low-stock alerts. Loading state shows skeleton pulsing placeholders.
2. Inventory: Product catalog grid with card layout, filter chips, search bar, stock badges
3. Orders: Order list with status filters, expandable cards with total and actions
4. Customers (CRM): Customer list with profile detail panel, segment badges, order history
5. Analytics: Revenue trend/forecast combo chart (green lines), top items bar chart, low-stock signals
6. Settings: Shop configuration form

COLOR PALETTE (WhatsApp-inspired):
- Primary teal: #128c7e (buttons, active states, brand elements)
- Darker teal: #0d7a6e (hover states)
- Light green accent: #dcf8c6 (badges, highlights, chat outbound bubbles)
- Background gray: #f0f2f5 (page background)
- Chat background: #ece5dd (WhatsApp preview panel)
- Dark text: #1a1a2e (headings, body text)
- Muted text: #4a4a4a (secondary text, descriptions)
- White: #ffffff (cards, search inputs, chat inbound bubbles)
- Success dot: #60b860 (live indicator)
- Soft green bg: #f0faf0 (success/paid states)

DESIGN PATTERNS:
- Cards: Rounded-2xl (1rem), white bg, subtle border (#128c7e/8 opacity), soft shadow
- Buttons: Rounded-full, primary teal bg, clean typography
- Badges: Rounded-full, small text, colored bg based on segment/status
- Typography: Sora for display/headings, Manrope for body text
- Charts: Recharts with green color scheme (#128c7e lines, #dcf8c6 grid)
- Form inputs: Rounded-xl, clean borders, teal focus ring
- WhatsApp chat bubbles: Rounded-2xl, #ffffff inbound, #dcf8c6 outbound, #ece5dd bg
- Live indicator: Pulsing green dot (#60b860)
- Mobile responsive: All grids adapt from 1→2→3→4 columns via sm:/md:/lg:/xl: breakpoints

FIGMA / DESIGN REFERENCES:
- Stakent Crypto Dashboard by Awsmd on Dribbble (for KPI card layout and dashboard structure)
- WhatsApp official color palette (teal #075e54, #128c7e, light green #dcf8c6, chat bg #ece5dd)

KEY TAILWIND CLASSES USED:
- Rounded shapes: rounded-full (nav, buttons, badges), rounded-[1.4rem] (cards), rounded-xl (inputs)
- Shadows: shadow-sm (cards), shadow-pill (navbar)
- Layout: max-w-7xl (content width), inline-flex (pill nav), grid with responsive cols
- Transitions: transition, hover:bg-*, active:scale-[0.99]
- Fonts: font-display (Sora), font-sans (Manrope)

The frontend should feel professional, accessible, and tailored for Indian small business owners — not like a generic SaaS dashboard. Every design decision should prioritize clarity and calmness over data density.
